# D-Done Eval — ADK Regression Tests

Offline evaluation harness for the DD pipeline extractors and synthesis agent.
Uses [ADK Eval](https://google.github.io/adk-docs/evaluate/) with an LLM judge
(`final_response_match_v2`, threshold 0.8).

> **Eval is offline** — it does NOT run during production DD checks.  
> It runs in CI (or manually) against frozen test documents and committed golden
> datasets to catch regressions when prompts or models change.

---

## Directory structure

```
eval/
  config.py                    ← reads EVAL_GCS_BUCKET / EVAL_PROJECT_NAME from env
  test_config.json             ← eval criteria (final_response_match_v2: 0.8)
  capture_golden.py            ← step 1: run pipeline → write .test.json golden files
  conftest.py                  ← pytest sys.path setup
  test_extractors.py           ← step 2: regression tests (pytest / CI)

  tabu_eval_agent/
    __init__.py                ← agent = tabu extractor + inject_pdfs callback
    tabu.test.json             ← golden dataset (generated by capture_golden.py)

  agreement_eval_agent/
    __init__.py
    agreement.test.json

  company_docs_eval_agent/
    __init__.py
    company_docs.test.json

  zero_report_eval_agent/
    __init__.py
    zero_report.test.json

  synthesis_eval_agent/
    __init__.py                ← agent = SequentialAgent(main + details)
    synthesis.test.json        ← pre-populated with extractor outputs in session state
```

---

## Step 0 — Upload test documents

Upload a representative set of PDFs to the frozen eval bucket:

```bash
       gsutil -m cp *.pdf gs://d-done-eval/test-docs/<project_name>/
```

> Files in this path must **never change** — they are the ground-truth inputs.

---

## Step 1 — Capture golden datasets

Run the capture script once after verifying the pipeline produces correct output:

```bash
cd backend

export GOOGLE_CLOUD_PROJECT=<your-gcp-project>
export GOOGLE_GENAI_USE_VERTEXAI=TRUE
export EVAL_GCS_BUCKET=d-done-eval
export EVAL_PROJECT_NAME=<project_name>   # folder you uploaded to

python eval/capture_golden.py
```

The script:
1. Lists PDFs from `gs://d-done-eval/test_docs/<project_name>/`
2. Runs the classifier → builds `doc_classification`
3. Runs each of the four priority extractors (tabu, agreement, company_docs, zero_report)
4. Runs synthesis with all extractor outputs pre-loaded in session state
5. Writes one `.test.json` per harness

**Review each `.test.json` carefully before committing** — these become the
regression ground truth.

---

## Step 2 — Run regression tests

```bash
cd backend

export GOOGLE_CLOUD_PROJECT=<your-gcp-project>
export GOOGLE_GENAI_USE_VERTEXAI=TRUE

# All eval tests
pytest eval/test_extractors.py -v

# Single harness
pytest eval/test_extractors.py::test_tabu_extractor -v
```

Tests skip automatically if the golden `.test.json` does not exist yet.

---

## How eval works

```
.test.json (frozen input + golden output)
         │
         ▼
  ADK Runner runs agent
  with session_input.state pre-seeded
         │
         ▼
  actual output ◄──── before_model_callback injects matching PDFs from GCS
         │
         ▼
  final_response_match_v2 (LLM judge, threshold 0.8)
         │
  PASS / FAIL
```

For **extractors**: the `before_model_callback` reads `doc_classification` from
session state and injects only the matching PDFs — exactly as in production.

For **synthesis**: all extractor outputs are pre-loaded in `session_input.state`
so the test runs without re-executing any extractors.

## CI integration

Add this step to your GitHub Actions workflow (trigger on PRs that touch
`extractors/**` or `synthesis/**`):

```yaml
- name: Run eval regression tests
  working-directory: backend
  env:
    GOOGLE_CLOUD_PROJECT: ${{ secrets.GCP_PROJECT }}
    GOOGLE_GENAI_USE_VERTEXAI: "TRUE"
  run: |
    pytest eval/test_extractors.py -v --tb=short
```

## Adding more extractors

To add a harness for a new extractor (e.g. `signing_protocol`):

1. Create `eval/signing_protocol_eval_agent/__init__.py` mirroring the other harnesses.
2. Add an entry to `EXTRACTOR_CONFIGS` in `capture_golden.py`.
3. Add a `test_signing_protocol_extractor` function to `test_extractors.py`.
4. Re-run `capture_golden.py` to generate the golden dataset.
